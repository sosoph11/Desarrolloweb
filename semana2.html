Qué es una página web

Definición:

  • Documento de texto que usa etiquetas para:
    • Modificar presentación
    • Incluir imágenes
    • Crear enlaces
    • Dar significado al contenido
  • Etiquetas entre signos de menor y mayor (< >)
  • Etiquetas en pares (apertura y cierre), algunas no requieren cierre (marcas vacías: <br>, <img>, <hr>)
  • Navegadores interpretan etiquetas para mostrar el diseño
  • Recomendación: usar hojas de estilo para la apariencia
  • Etiquetas pueden anidarse; los saltos de línea y espacios en blanco son ignorados
  • Etiquetas brindan significado semántico, no solo estilo visual

Atributos:

  • Pueden tener uno o varios atributos en la etiqueta de apertura
  • Ejemplo: <etiqueta atributo="valor" atributo2="valor2">
  • Valores de atributos entre comillas (no necesario si no hay espacios)
  • Se recomienda usar hojas de estilo para modificar el aspecto visual

Comentarios:

  • No visibles en el navegador, salvo en el código fuente
  • Etiqueta: <!-- .... -->
  • Pueden estar dentro o fuera de las etiquetas

Página básica:

  • <!DOCTYPE ...>: Obligatoria, indica el tipo de documento (HTML5: "html")
  • <html> ...</html>: Engloba todo el documento
  • Dividido en:
    • <head> ...</head>: Información de identificación/control, enlaces a hojas de estilo, <title> ...</title> (obligatoria), <link> (referencia a documentos como CSS), <script> (código JavaScript)
    • <body> ...</body>: Contenido visible en la ventana del navegador

Sintaxis de HTML:

  • Define reglas para escribir elementos correctamente
  • Documentos con errores de sintaxis son inválidos, pero navegadores intentan corregir
  • Recomendación: validar y corregir documentos HTML
  • XHTML en HTML5: basada en XML, más estricta que HTML, que es más flexible

Generalidades:

  • < solo para iniciar etiquetas; para mostrar <, usar &lt;
  • Nombres de elementos, atributos y valores en minúsculas (pueden ir en mayúsculas)
  • Nombre del elemento sin espacios después de <
  • Nombre en etiqueta de cierre debe coincidir con apertura
  • Elemento raíz <html>...</html> (generalmente único, pero puede omitirse)
  • Elementos deben estar completamente contenidos dentro de otros (no cruzados)
  • Elementos no vacíos tienen etiquetas de cierre (algunas circunstancias permiten omitirlas)
  • Elementos vacíos no tienen etiqueta de cierre, pueden acabar en /> (ejemplos: <img>, <br>, <hr>)
  • Atributos solo en la etiqueta de apertura; valores entre comillas (si tienen espacios)
  • Valor del atributo id debe ser único en el documento

Atributos en HTML

  • Cualquier etiqueta HTML puede tener uno o más atributos separados por espacios.
  • El valor se asigna con = (excepto atributos booleanos).
  • Los valores deben ir entre comillas si contienen espacios, pero no es necesario para un solo valor.

Atributos Comunes:

  • class:
    • Define clases de elementos.
    • Usado en hojas de estilo y selectores.
  • id:
    • Identifica de manera única un elemento en un documento.
    • No pueden repetirse en la misma página.
    • Solo puede tener un valor sin espacios.
  • style:
    • Define propiedades de estilo directamente en el elemento.
    • Se desaconseja su uso; es preferible usar hojas de estilo externas.

Clases en HTML

Asignación de Clases:

  • Se usa el atributo class.
  • Nombre de la clase: caracteres ingleses, números, guiones y subrayados (no puede empezar con número o guión).

Aplicación de Clases:

  • Las reglas de estilo se aplican a los elementos con esa clase.
  • Los navegadores distinguen entre mayúsculas y minúsculas; se recomienda usar minúsculas.
  • Un elemento hereda propiedades tanto de la clase como de la etiqueta general.

Combinación de Clases:

  • Se pueden asignar varias clases a un elemento, separadas por espacios.
  • El orden de las clases en el atributo class no importa; el orden de las reglas en la hoja de estilo sí.
  • La última definición en la hoja de estilo prevalece en caso de conflictos.

Selección de Elementos con Múltiples Clases:

  • Se pueden escribir selectores para elementos con varias clases.

Uso de la Misma Clase en Diferentes Elementos:

  • Se puede asignar la misma clase a diferentes elementos.
  • Cada elemento puede tener propiedades distintas o iguales, según la definición de la clase.

Formularios en HTML

Etiqueta <form>:

  • Delimita un formulario.

Atributos importantes:

  • action: Nombre del agente que procesa los datos (ej. script PHP).
  • method: Método para enviar datos. Valores posibles:
    • get: Añade datos a la URL.
    • post: Envía datos de forma separada.
  • Si method no está definido, se usa get por defecto.

Controles en Formularios:

  • Etiquetas principales:
    • <input>, <button>, <select>, <optgroup>, <option>, <textarea>.
    • Estructura: <fieldset>, <legend>.
    • Accesibilidad: <label>.

Atributos de Controles:

  • type: Define el tipo de control (ej. text, password, number).
  • name: Identifica el control. No se deben repetir (excepto en botones radio).
  • value: Valor inicial del control. Puede mostrarse al usuario o no (excepciones: archivo, imagen).
  • required: Indica si un control es obligatorio.
  • placeholder: Texto que aparece en el control de texto antes de que el usuario escriba.
  • size: Longitud de los controles de texto.
  • maxlength y minlength: Longitud máxima y mínima permitida para entradas de texto.
  • disabled: Desactiva el control.

Controles Específicos:

  • Botones:
    • <input>: Solo texto, definido por value.
    • <button>: Permite contenido HTML (ej. imágenes).
  • Caja de texto de contraseña: <input type="password"> (muestra puntos en lugar de texto).
  • Número: <input type="number"> (con min, max, step para restricciones numéricas).
  • Casilla de verificación: <input type="checkbox">.
  • Botón radio: <input type="radio"> (múltiples botones con el mismo name forman un grupo).
