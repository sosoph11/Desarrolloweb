Introducción a JavaScript

JavaScript es un lenguaje de programación que se utiliza principalmente para crear interactividad y dinamismo en las páginas web. Permite manipular elementos HTML y CSS, responder a eventos del usuario y realizar tareas asincrónicas como la carga de datos desde un servidor.

Etiqueta <script>

Contiene código JavaScript que se ejecuta automáticamente al procesar la etiqueta en el navegador.

Script Externo

Se puede incluir código JavaScript desde un archivo separado usando el atributo src de la etiqueta <script>. Esto organiza y modulariza el código para facilitar su mantenimiento.

<script src="/ruta/al/script.js"></script>

  • Ruta Absoluta: /ruta/al/script.js apunta al archivo desde el directorio raíz.
  • Ruta Relativa: src="script.js" apunta al archivo en el mismo directorio que la página web.
  • Ruta en la Web: src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/3.2.0/lodash.js" carga un archivo JavaScript desde la web.

Vincular Múltiples Scripts

Se pueden incluir varios archivos de script:

<script src="/js/script1.js"></script>

<script src="/js/script2.js"></script>

¿Dónde incluir nuestro script?

La etiqueta <script> puede colocarse en el <head> o en el <body>, y antes, después o en medio del contenido. La ubicación ideal depende de las características del script y del sitio web para optimizar la carga y ejecución.

El punto y coma se puede omitir en algunos casos utilizando saltos de línea. Aunque JavaScript puede interpretar un punto y coma implícito en los saltos de línea, es recomendable especificarlo de manera explícita como una buena práctica.

Para hacer un comentario de una sola línea, usa dos barras paralelas //. Para comentarios de múltiples líneas, comienza con /* y termina con */.

Funciones y Variables

Las funciones son fundamentales para ejecutar acciones similares en diferentes partes del script. Por ejemplo, podemos usar funciones para mostrar un mensaje amigable cuando un visitante se autentica, cierra sesión u realiza otras acciones. En esencia, las funciones permiten organizar nuestro código en "bloques" reutilizables.

Variables

Antes de ECMAScript 6, se usaba var para definir variables, pero esta actualización introdujo let y const para definir variables y constantes, respectivamente.

Ámbito de las Variables

  • Variables Locales: Declaradas dentro de un bloque y accesibles solo dentro de ese bloque (delimitado por llaves {}).
  • Variables Globales: Declaradas fuera de cualquier bloque, accesibles tanto dentro como fuera de los bloques.

DOM (Document Object Model)

Es una interfaz de programación que representa un documento HTML o XML como una estructura jerárquica de nodos, donde cada nodo corresponde a un elemento, atributo o texto en el documento. Permite a los desarrolladores acceder y manipular el contenido, estructura y estilo de las páginas web de manera dinámica mediante programación.

Árbol de Nodos

  • Raíz: Nodo document.
  • Tipos de Nodo:
    • Document: Nodo raíz.
    • Element: Representa etiquetas HTML.
    • Attr: Representa atributos HTML.
    • Text: Contiene texto dentro de una etiqueta.

Acceso a Nodos

Métodos:

  • getElementsByTagName()
  • getElementsByName()
  • getElementById()
  • getElementsByClassName()

Propiedades CSS: Usar style para acceder y modificar.

Atributos HTML: Acceder usando el nombre del atributo.

Creación y Eliminación de Nodos

  • Crear: document.createElement(), document.createTextNode(), appendChild().
  • Eliminar: remove().

Trabajar con Clases

Propiedad classList con métodos:

  • add: Añade una clase.
  • remove: Elimina una clase.
  • toggle: Alterna la presencia de una clase.
  • contains: Verifica la existencia de una clase.

JSON (JavaScript Object Notation)

Es un formato de datos basado en texto que sigue la sintaxis de objetos de JavaScript. Aunque es similar a los objetos literales de JavaScript, puede usarse en diferentes entornos de programación. JSON es útil para transmitir datos a través de redes y debe convertirse a objetos de JavaScript para su uso. Los datos JSON se almacenan en archivos de texto con la extensión .json.

Estructura del JSON

Formato: Similar a objetos literales JavaScript.

Tipos de datos básicos:

  • Cadenas
  • Números
  • Arreglos
  • Booleanos
  • Literales de objeto

Función: Permite construir jerarquías de datos.

Conversiones entre objetos y texto

  • Objetos a JSON: Usar JSON.stringify(objeto) para convertir un objeto JavaScript a una cadena JSON.
  • JSON a objetos: Usar JSON.parse(cadenaJSON) para convertir una cadena JSON a un objeto JavaScript.

Estos métodos están disponibles en los navegadores y son comunes en el desarrollo web.

HTTP

HTTP (Hypertext Transfer Protocol) es un protocolo de comunicación utilizado para la transferencia de datos en la web. Permite la transmisión de información entre un cliente (como un navegador web) y un servidor. HTTP define cómo se envían y reciben los mensajes, incluyendo las solicitudes del cliente y las respuestas del servidor, facilitando el intercambio de recursos como páginas web, imágenes y otros archivos.

Método GET

  • Propósito: Solicitar datos del servidor.
  • Uso: Obtener información sin modificar el estado del servidor.
  • Datos: Los datos se envían en la URL como parámetros de consulta (query parameters).
  • Ejemplo: GET /productos?id=123
  • Características: La información es visible en la URL y tiene un límite de tamaño.

Método POST

  • Propósito: Enviar datos al servidor para crear o actualizar recursos.
  • Uso: Modificar el estado del servidor, como enviar formularios.
  • Datos: Los datos se envían en el cuerpo de la solicitud, no en la URL.
  • Ejemplo: POST /crear-producto con datos en el cuerpo de la solicitud.
  • Características: Los datos no son visibles en la URL y no tienen un límite de tamaño estricto.

Fetch (Comunicación con el servidor)

Fetch es una función de JavaScript que se utiliza para realizar solicitudes HTTP a un servidor de manera asíncrona. Permite enviar y recibir datos sin necesidad de refrescar la página, facilitando la comunicación entre un cliente (como un navegador) y un servidor. Fetch devuelve una promesa que se resuelve con un objeto "Response", el cual proporciona métodos para acceder al cuerpo de la respuesta en distintos formatos, como texto o JSON.

Esto permite realizar tareas como:

  • Obtener datos de usuario
  • Cargar listados de productos
  • Recibir notificaciones

AJAX (Asynchronous JavaScript And XML) es un término para estas solicitudes, aunque hoy en día se usa más JSON que XML.

Sintaxis básica:

let promise = fetch(url, [options]);

  • url: URL del recurso.
  • options: Opcionales, como el método HTTP y encabezados.

Procesos:

  1. Fase 1: fetch devuelve una promesa que se resuelve con un objeto "Response" cuando el servidor responde. Se puede verificar el estado HTTP con "status" y "ok".
  2. Fase 2: Analizar el cuerpo de la respuesta usando métodos como "response.text()" para texto o "response.json()" para JSON.

Uso de await

Await pausa la ejecución hasta que la promesa se resuelve o rechaza, y retorna el valor de la promesa. Solo se usa dentro de funciones async.
